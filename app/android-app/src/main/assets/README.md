# 拼豆像素画设计工具

这是第一阶段电脑端原型，聚焦素材图片和外部图纸转实体色块像素画的核心流程。

## 当前能力

- 在“生成图纸”页导入素材图片并生成像素图。
- 左侧会显示当前运行模式，区分本机服务版和静态版。
- 左侧菜单栏和生成图纸上传素材面板都可收起，为像素画预览留出更大空间。
- 生成图纸上传素材栏、色块统计栏、色块库编辑区和色块列表均支持独立滚动。
- 生成结果会先进入候选状态，可重新生成、查看候选历史，满意后再采纳到图库。
- 支持主体居中、完整画面、手动裁剪三种取样方式；手动裁剪可直接在预览图拖动取景并缩放。
- 素材图本地像素降采样，并映射到当前色块库。
- 使用 Lab 色彩空间和 Delta E 距离，把像素映射到实体色块库。
- 色差风险提示，提醒色块库覆盖不足的区域，并可一键把建议色加入色块库。
- 色块库管理：可用色块上限、编号、名称、色盘取色、色彩地图选点、RGB 输入、启用/禁用、导入模板下载。
- 色块库库存：记录每种豆子的当前库存，支持单个入库/扣减、批量设定、批量补充、批量扣减，库存低于 100 个时提醒补货。
- 色块库可导入、导出，并支持软删除。
- 像素画侧边栏统计、点击同名同色高亮隔离、一键全局替换、撤销替换。
- 支持生成后手工编辑：逐格画笔、连通区域填充、画布取色、坐标提示。
- 生成图纸页支持解析外部图纸图片：按上半部分网格内文字色号逐格 OCR 还原，确认后可采纳到图库；底部调色板不会入库。
- 图片预览区支持触摸板双指缩放、双指滚动平移，以及空格或中键拖动画布。
- 编号图层语义缩放：格子大于 20px 时才显示编号。
- 采纳作品到个人图库，保存矩阵和色块快照，可导入/导出/删除作品。
- 拼装页可从图库作品进入，在像素图上标记现实中已拼好的格子，并按色块扣减剩余数量。
- 拼装页完成标记会同步扣减对应色块库存，取消标记会把库存加回。
- 拼装页支持点击色块高亮同色块，定位到该色块下一个未完成格，并自动放大居中显示编号。
- 首页看板汇总图库、拼装进度和低库存补货提醒。
- AI 设置可维护多个模型配置，保存供应商、模型名、自定义端点、API Key 和卡通化提示词，并选择当前启用配置。
- AI 设置页可运行诊断，查看当前运行模式、数据目录、数据文件和启用模型。
- AI 设置页可打开数据目录，并导出或恢复包含色块库、图库、色块上限和 AI 设置的完整备份。
- 双击启动本机服务后，真实照片可优先调用云端图像模型做卡通化，失败会回退到本地预处理。
- 导出带编号 PNG、色块用量 CSV、坐标矩阵 CSV、分区打印版 HTML。

## 运行方式

推荐在 macOS 上双击 `拼豆设计.app`。也可以双击 `打开拼豆设计.command`。

Windows 可双击 `打开拼豆设计.bat`。Linux 可运行 `scripts/launch-unix.sh`。

这些入口都会启动本机服务并打开 `http://127.0.0.1:4173`，数据会保存到 `data/` 文件夹。

也可以直接在浏览器打开 `index.html` 运行静态版。静态版不启动本机服务，数据保存到浏览器本地存储，云端 AI 调用会自动使用本地预处理替代。

如果之前用静态版保存过色块或作品，改用 `拼豆设计.app` 打开后，进入“AI 设置”点击“迁移静态版数据”即可搬到 `data/` 文件夹。

打开后可先看左侧运行状态：显示“Node 本机服务”或“Python 后备服务”时，数据会保存到 `data/` 文件夹；显示“静态版”时，数据保存在当前浏览器本地存储。

本机服务优先使用 Node.js 20 或更高版本。如果双击应用找不到 Node.js，会自动尝试 Python 3 后备服务；Python 后备服务支持本机数据保存和非 AI 功能，云端 AI 调用需要 Node.js 服务。Node 和 Python 都不可用时会自动打开静态版。

最快试用：双击 `拼豆设计.app`，进入“生成图纸”页，导入素材图片，点击“生成像素画”，再点击“导出生产文件”。

详细步骤见 `使用手册.md`。

## 云服务器部署依赖

当前项目已可按私有 Web 应用部署。图纸解析功能需要服务器安装 Tesseract OCR，否则系统会拒绝解析并提示缺少 OCR 引擎，避免用颜色匹配生成偏差很大的作品。

Ubuntu / Debian：

```bash
sudo apt update
sudo apt install -y tesseract-ocr
```

macOS 本机测试：

```bash
brew install tesseract
```

安装后重启 Node 服务，再在“生成图纸”页点击“解析图纸”。解析会严格以每个格子里的文字色号为准，颜色只做辅助校验。

## 正式桌面安装包

第一版正式安装包使用 Electron 打包，目标为 macOS 和 Windows。安装包内会自带运行环境，用户不需要单独安装 Node.js 或 Python。

首次启动时，软件会把 `defaults/` 中的默认数据复制到系统用户数据目录：

- macOS：`~/Library/Application Support/拼豆设计`
- Windows：`%APPDATA%/拼豆设计`

默认数据来自当前配置：221 个色块、色块上限 221，以及当前 8 个图库作品作为示例作品。示例作品会保留图案内容，但不会带入当前拼装进度。默认 AI 设置为本地预处理，不包含 API Key。

发布前生成默认数据：

```bash
npm run build:defaults
```

本地调试桌面版：

```bash
npm install
npm run dev:desktop
```

生成安装包：

```bash
npm run package:mac
npm run package:win
```

生成结果会输出到 `dist/`。第一版未配置代码签名，macOS 和 Windows 可能会出现安全提示；后续商业版可补充 Apple Developer ID、Windows 代码签名证书和自动更新。

## AI 配置

- OpenAI 模式默认调用图像编辑接口，把真实照片先转为更适合像素化的卡通图。
- 自定义模型端点需要接收 `{ imageDataUrl, model, prompt }`，并返回 `imageDataUrl` 或 `b64_json`。
- 可以新增多个模型配置，并在“启用模型配置”中切换当前用于真实照片卡通化的模型。
- API Key 在本机服务模式下保存到 `data/settings.json`；完整备份不会明文导出 API Key。
- 点击“运行诊断”可确认当前是 Node 服务、Python 后备服务还是静态版。
- 点击“打开数据目录”可查看本机保存文件；点击“备份全部数据”会导出完整 JSON 备份，其中 API Key 只保留是否已设置的状态，不会明文导出。
- 点击“恢复完整备份”会替换当前色块库、图库和 AI 设置，适合换电脑或从静态版切换到服务版后恢复整套数据。

## 生产导出

- 编号 PNG 适合快速查看整图。
- 色块用量 CSV 用于备料统计。
- 坐标矩阵 CSV 用于按行列核对。
- 分区打印版 HTML 会把大图自动拆成 32 x 32 区块，每个区块单独带坐标和用量，可直接打印或另存为 PDF。

## 功能覆盖

- 图片输入：素材图片、外部图纸解析。
- 转换链路：素材图片直接本地像素化；外部图纸按格内色号解析还原。
- 色块库：颜色、编号、名称、可用上限、导入导出、软删除。
- 库存管理：当前库存、补充入库、拼装扣减、低库存提醒。
- AI 管理：多个模型配置、启用配置、OpenAI/自定义/本地模式。
- 生成交互：重新生成、候选历史、采纳到图库。
- 图库展示：作品快照、编号图层、色块统计、同名同色高亮。
- 拼装跟进：已拼标记、剩余数量扣减、同色高亮、下一个未完成色块定位。
- 编辑工具：画笔、填充、取色、全局替换、撤销、触摸板缩放和平移。
- 生产输出：整图、用量清单、坐标矩阵、分区打印单。
- 数据维护：图库备份、完整数据备份和恢复、数据目录打开、静态版数据迁移。

## 自检

运行 `npm run check` 会检查前端脚本、Node 服务端脚本、Python 后备服务、跨平台启动器、DOM 控件绑定、核心数据结构、UI 收起/缩放/平移交互、本机服务 API 读写逻辑、AI 链路回退/响应解析、候选结果采纳到图库，以及拼装页标记、扣减、定位和进度保存工作流。
