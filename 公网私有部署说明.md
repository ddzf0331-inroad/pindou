# 拼豆设计公网私有部署说明

适用场景：长期只给一个人通过浏览器使用。部署后用户访问一个 HTTPS 地址，输入访问密码即可使用，色块库、图库、库存和拼装进度都会保存到服务器数据目录。

## 一、推荐服务器

选择一台轻量云服务器即可。

- 系统：Ubuntu 22.04 或 24.04
- 配置：1 核 2G 起步
- 磁盘：40G 起步
- 端口：开放 80、443

## 二、上传项目

把整个项目目录上传到服务器，例如：

```bash
/opt/pindou
```

进入项目目录：

```bash
cd /opt/pindou
npm install
```

## 三、准备数据目录

建议把正式数据放到项目外，避免升级代码时误删。

```bash
sudo mkdir -p /data/pindou
sudo chown -R $USER:$USER /data/pindou
```

第一次启动时，服务会自动把 `defaults/` 里的默认色块库、图库、色块上限和默认 AI 设置复制到这个目录。已有文件不会被覆盖。

## 四、设置访问密码并启动

把下面的密码换成你自己发给用户的密码：

```bash
export PIXEL_TOY_PASSWORD="这里改成你的访问密码"
export PIXEL_TOY_DATA_DIR="/data/pindou"
export PORT=4173
npm run start:private-web
```

启动后服务会监听：

```text
0.0.0.0:4173
```

如果暂时没有域名，可以先访问：

```text
http://服务器公网IP:4173
```

正式长期使用建议继续配置域名和 HTTPS。

## 五、用 PM2 常驻运行

安装 PM2：

```bash
npm install -g pm2
```

启动：

```bash
PIXEL_TOY_PASSWORD="这里改成你的访问密码" \
PIXEL_TOY_DATA_DIR="/data/pindou" \
PORT=4173 \
PIXEL_TOY_PUBLIC_MODE=1 \
HOST=0.0.0.0 \
pm2 start server.js --name pindou
```

设置开机自启：

```bash
pm2 save
pm2 startup
```

查看状态：

```bash
pm2 status
pm2 logs pindou
```

## 六、配置域名和 HTTPS

安装 Nginx：

```bash
sudo apt update
sudo apt install -y nginx
```

新增站点配置：

```nginx
server {
  listen 80;
  server_name 你的域名;

  client_max_body_size 50m;

  location / {
    proxy_pass http://127.0.0.1:4173;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

启用配置后重载 Nginx。再用 Certbot 配 HTTPS：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d 你的域名
```

完成后用户访问：

```text
https://你的域名
```

## 七、备份数据

最重要的数据在：

```text
/data/pindou
```

建议每天备份一次：

```bash
mkdir -p /data/pindou-backups
tar -czf /data/pindou-backups/pindou-$(date +%F).tar.gz /data/pindou
```

可加入定时任务：

```bash
crontab -e
```

加入：

```cron
0 3 * * * tar -czf /data/pindou-backups/pindou-$(date +\%F).tar.gz /data/pindou
```

## 八、升级应用

升级代码前先备份：

```bash
tar -czf /data/pindou-backups/before-upgrade-$(date +%F-%H%M).tar.gz /data/pindou
```

然后替换 `/opt/pindou` 里的项目文件，再执行：

```bash
cd /opt/pindou
npm install
pm2 restart pindou
```

注意：只要 `PIXEL_TOY_DATA_DIR` 仍然指向 `/data/pindou`，升级不会覆盖用户已有数据。

## 九、安全注意事项

- 必须设置 `PIXEL_TOY_PASSWORD`，不要裸露公网。
- 不要把 `data/settings.json` 里的 API Key 发给别人。
- 公网模式下页面会隐藏“打开数据目录”和“迁移静态版数据”。
- 如果密码泄露，修改 `PIXEL_TOY_PASSWORD` 后重启服务即可。
- 第一版是单用户私有部署，不适合多人同时共用不同账号。
